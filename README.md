ТЕХНИЧЕСКОЕ ЗАДАНИЕ НА РАЗРАБОТКУ И ВНЕДРЕНИЕ ПРОГРАММНОГО ПРОДУКТА
1.	ОБЩИЕ СВЕДЕНИЯ
1.1.	Полное наименование ИС: Информационная система «GBnails — Telegram мини-приложение записи к мастеру маникюра».
1.2.	Шифр темы: ИС-GBnails-TG-01.
1.3.	Предприятие-разработчик системы: СМЗ «Будыкин Н.К.».
1.4.	Предприятие-заказчик системы: индивидуальный мастер маникюра.
1.5.	Система создается на основании технического задания (ТЗ): Настоящий документ определяет цели, задачи, требования и порядок разработки.
1.6.	Плановый срок начала работ: 18.09.2025.
1.7.	Плановый срок окончания работ: 01.12.2025.
1.8.	Автоматизируемая система создается на современной основе с использованием платформы Telegram Mini Apps, серверного API Python FastAPI, базы данных PostgreSQL и веб-технологий HTML, CSS, JS (React).
1.9.	Порядок оформления и предъявления заказчику результатов работы: После создания первой версии продукта, содержащей основные функции (запись на услуги, добавление времени, управление клиентами), система предоставляется заказчику для тестирования. После тестирования оформляется акт приёма-передачи.
2.	НАЗНАЧЕНИЕ И ЦЕЛИ СОЗДАНИЯ СИСТЕМЫ
2.1.	Вид автоматизируемой деятельности: Автоматизация процессов записи клиентов к мастеру маникюра, управления расписанием и клиентской базой.
2.2.	Перечень автоматизируемых процессов:
2.2.1.	Для клиента:
	Запись клиента на процедуру через Telegram-мини-приложение.
	Выбор услуги из предложенного списка.
	Выбор свободного тайм-слота для записи
	Напоминания о записях в телеграмм-боте. 
	Подтверждение записи за день до визита. 
2.2.2.	Для мастера:
	Добавление свободных временных слотов.
	Ведение клиентской базы (ФИО, контакты, история посещений, заметки про клиента).
	Формирование календаря-расписания.
	Уведомления о записях в телеграмм-боте. 
2.3.	Наименование и значение показателей, которые будут достигнуты:
	Сокращение времени на организацию записей (в 2–3 раза).
	Уменьшение количества ошибок (двойных записей, потерянных клиентов).
	Увеличение удобства взаимодействия клиента с мастером.
	Повышение лояльности клиентов за счёт удобства записи и уведомлений.
3.	ХАРАКТЕРИСТИКИ ОБЪЕКТА АВТОМАТИЗАЦИИ
Краткие сведения о предприятии: Предприятие — индивидуальный мастер маникюра, оказывающий услуги по уходу за руками и ногтями. Ведение записей осуществляется вручную или через мессенджеры, что требует автоматизации.
4.	ТРЕБОВАНИЯ К ИС
4.1.	Требования к системе в целом:
	Доступность сервиса 24/7 через Telegram.
	Авторизация пользователей через Telegram-аккаунт.
	Простота интерфейса и удобство использования.
	Поддержка мобильных и десктопных устройств.
4.2.	Требования к программному обеспечению ИС:
	Серверная часть: Python FastAPI.
	База данных: PostgreSQL.
	Веб-клиент: HTML, CSS, JS (React).
	Интеграция с Telegram Bot API и Telegram Mini Apps API.
Требования к техническому обеспечению АС:
	Размещение серверной части на хостинге или VPS.
	Минимальные требования: 2 ГБ ОЗУ, 1 CPU, 20 ГБ SSD.
	Клиентская часть доступна в Telegram без установки дополнительного ПО.
